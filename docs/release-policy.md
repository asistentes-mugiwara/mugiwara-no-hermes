# 🚀 Release Policy

## Modelo de versionado

Este proyecto usa **versionado semántico editorial**.

- **MAJOR** → reestructuración fuerte del escaparate o del relato arquitectónico
- **MINOR** → nuevas secciones, ejemplos, diagramas o capacidad documental pública
- **PATCH** → correcciones, aclaraciones, redacciones de seguridad y mejoras de copy

## 📦 Cuándo sale una release

Se crea una release cuando el escaparate gana una porción nueva y real de valor explicativo.

No hace falta esperar a que haya “mucho volumen”.
Hace falta que haya **mejor comprensión pública**.

## 📝 Qué debe explicar cada release

Cada release debería contar:

- qué se añadió
- qué se aclaró
- qué se retiró o redujo por seguridad
- por qué esta release merece existir

## 🎭 Regla editorial

Las release notes pueden tener voz, carisma y algo de épica.
Pero deben seguir siendo útiles para alguien que quiere entender el cambio en 30 segundos.

Nada de humo legendario sin contenido. El Rey de los Piratas quizá improvise; un changelog bueno no.
