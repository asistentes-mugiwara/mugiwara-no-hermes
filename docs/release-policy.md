# Release Policy

## Versioning model

This project uses **editorial semantic versioning**.

- **MAJOR**: major restructuring of the public narrative or architecture explanation
- **MINOR**: new sections, examples, diagrams or public-safe capabilities
- **PATCH**: corrections, clarifications, redactions and documentation fixes

## Release rule

A release is created when the public showcase gains a meaningful new slice of explanatory value.

## Release contents

Each release should explain:
- what was added
- what was clarified
- what was removed or reduced for safety
- why the release matters
