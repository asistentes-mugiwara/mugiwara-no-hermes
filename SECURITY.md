# Security Policy

## Scope

This repository is a **public, sanitized showcase** of a real multi-agent system.

## Reporting a problem

If you think something sensitive, dangerous or overexposed has been published here by mistake:

1. do **not** open a public issue with the sensitive material
2. contact the maintainer privately through the repository owner account if possible
3. describe the path and the type of exposure, without repeating the secret itself

## Policy

- sensitive material should be removed quickly
- security-driven redactions are acceptable and expected
- public completeness is always secondary to operational safety
